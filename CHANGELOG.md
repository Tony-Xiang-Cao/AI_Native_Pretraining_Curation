# Changelog

All notable changes to `autocurate` are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] - 2026-06-14

Initial release. `autocurate` operationalizes a data-governance theory — four
directions of *traceable sources*, *assessable quality*, *graded risk*, and
*divisible responsibility* — into an AI-native, self-verifying, self-improving
pretraining-data curation loop. The core is pure-stdlib, CPU-only, and
deterministic; LLM and visualization dependencies are optional extras.

### Added

- **`profile`** — corpus profiling via cheap heuristics plus robust-z
  (median/MAD) outlier detection, more resistant to contamination than
  mean ± k·sigma.
- **`extract`** — reference-free extraction-quality gates for HTML, OCR, and
  JSON sources, scoring documents without a gold reference.
- **`judge`** — wraps the sibling `judgecurate` package (optional `[judge]`
  extra) to provide the LLM-as-judge middle stage of the curation cascade.
- **`verify`** — the heart of the system: a mutation-oracle verification
  harness that corrupts known-good documents with a **held-out corruption
  vocabulary** (disjoint from the curation lexicon) so the oracle cannot be
  gamed, then measures whether gates catch the injected defects.
- **`hillclimb`** — verifier-gated self-improvement of curation rules: every
  candidate change must pass the mutation oracle before it is accepted,
  preventing reward over-optimization.
- **`agentloop`** — a harness-agnostic agent loop (`Routine` + `AgentHarness`)
  with a provenance ledger and EWMA/CUSUM drift detection for production
  monitoring.
- **`report`** — summary and figure reporting over curation runs.
- **`baselines`** — external reference-free baselines (gzip compression screen,
  Gopher/C4 heuristics, Alex & Burns dictionary/garbage ratio, `json.loads`-only)
  scored on the same mutation oracle.
- Seven reproducible experiments (**E1–E7**) with committed results under
  `results/*.json`, regenerated by `scripts/run_experiments.py` and plotted by
  `scripts/make_figures.py`:
  - **E1** — extraction-gate F1: hard-gate (HTML+OCR) floor 0.851 / upper 0.977
    (HTML 0.887/0.972, OCR 0.816/0.982); JSON is a parse oracle (1.0, separate).
  - **E2** — cascade routing against `judgecurate`'s offline heuristic judge:
    router-only macro-F1 0.433, judge-everything 0.653; 50% calls → 0.584
    (~69% of the gain). Optional real-LLM rerun via `scripts/run_llm_e2.py`.
  - **E3** — un-guarded recall hill-climbing destroys 51.7% of *held-out* clean
    data; un-guarded balanced-F1 is empirically safe (FPR 0.017) but unguaranteed;
    the guard gives a provable 0.0. Disjoint climb/eval document draws.
  - **E4** — drift monitoring over 8 streams: 0-day detection, 0.125 false
    alarms/stream, quality 0.885 → 0.582 on shift, ~5.5-day recovery.
  - **E5** — robust-z outlier F1 0.932 vs. mean ± k·sigma 0.889 (z=3.5; robust
    wins at 3/4 cutoffs).
  - **E6** — drift-triggered self-improvement: a gate blind to tag-leakage detects
    at F1 0.07; the verified loop adapts it to 0.98 (guard FPR 0); naive control
    over-flags (FPR 0.25).
  - **E7** — gates beat every external reference-free baseline on every modality.
- Statistics use a small-sample Student-t lower bound (not a normal/bootstrap
  approximation); the accept rule's FPR clause is exercised by a dedicated test.
- Optional extras: `[judge]` (`judgecurate>=0.3.0`), `[llm]` (`anthropic`,
  `openai`), `[viz]` (`matplotlib`), `[dev]` (`pytest`, `ruff`).
